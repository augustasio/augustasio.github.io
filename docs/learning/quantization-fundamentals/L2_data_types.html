Floating Points

FP32:

  • Sign: 1 bit
  • Exponent (range): 8 bit
  • Fraction (precision): 23 bit
  • Smallest: \(1.4 \times 10^{-45}\)
  • Largest: \(3.4 \times 10^{38}\)

BF16:

  • Sign: 1 bit
  • Exponent (range): 8 bit
  • Fraction (precision): 7 bit
  • Smallest: \(9.2 \times 10^{-41}\)
  • Largest: \(3.4 \times 10^{38}\)

FP16:

  • Sign: 1 bit
  • Exponent (range): 5 bit
  • Fraction (precision): 10 bit
  • Smallest: \(6.0 \times 10^{-8}\)
  • Largest: \(6.5 \times 10^{4}\)

Downcasting

# random pytorch tensor: float32, size=1000
tensor_fp32 = torch.rand(1000, dtype=torch.float32)

Note: As it is random, the values you get will be different from the video.

# first 5 elements of the random tensor
tensor_fp32[:5]
tensor([0.5020, 0.8735, 0.2707, 0.2318, 0.3678])
# downcast the tensor to bfloat16 using the "to" method
tensor_fp32_to_bf16 = tensor_fp32.to(dtype=torch.bfloat16)
tensor_fp32_to_bf16[:5]
tensor([0.5039, 0.8750, 0.2715, 0.2314, 0.3672], dtype=torch.bfloat16)
# Let's check the MSE (notice the automatic upcasting to float32)
((tensor_fp32 - tensor_fp32_to_bf16.float()) ** 2).mean(), ((tensor_fp32 - tensor_fp32_to_bf16) ** 2).mean()
(tensor(7.1364e-07), tensor(7.1364e-07))
# tensor_fp32 x tensor_fp32
m_float32 = torch.dot(tensor_fp32, tensor_fp32)
m_float32.item()
308.4287109375
# tensor_fp32_to_bf16 x tensor_fp32_to_bf16
m_bfloat16 = torch.dot(tensor_fp32_to_bf16, tensor_fp32_to_bf16)
m_bfloat16.item()
308.0

Mixed-precision

  • Do computation in smaller precision (FP16/BF16/FP8)
  • Store and update the weights in higher precision (FP32)
